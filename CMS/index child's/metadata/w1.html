Web Technology Units, Topics, and Objectives

This document is a for-discussion-purposes list of the topics that I submit should be covered in an upper-level computer science web technologies course. To flesh this out and in an attempt to follow the spirit of CC2001, I've grouped the topics into units and listed representative tasks (objectives) that I think students should be able to perform relative to the topics.
  1. Fundamental Web Concepts
    Topics:
    • HTTP
    • URIs
    • Web browsers
    • Web servers
    • HTML
    • CSS
    • Accessibility
    Learning objectives:
    1. Write syntactically correct HTTP messages and describe the semantics of common HTTP methods and header fields
    2. Discuss differences between URIs, URNs, and URLs, and demonstrate a detailed understanding of http-scheme URLs, both relative and absolute
    3. Describe the actions, including those related to the cache, performed by a browser in the process of visiting a Web address
    4. Install a web server and perform basic administrative procedures, such as tuning communication parameters, denying access to certain domains, and interpreting an access log
    5. Write a valid standards-conformant HTML document involving a variety of element types, including hyperlinks, images, lists, tables, and forms
    6. Use CSS to implement a variety of presentation effects in HTML and XML documents, including explicit positioning of elements
    7. Demonstrate techniques for improving the accessibility of an HTML document
  2. Client-side Programming
    Topics:
    • Scripting language characteristics
    • Language features (traditional and nontraditional)
    • Regular expressions
    • Event-driven programming
    • Document Object Model (DOM) API
    • De facto standard host objects
    • Debugging client-side programs
    Learning objectives:
    1. Give the distinguishing characteristic of scripting languages
    2. Discuss the reasons for and effects of nonstandard client-side scripting language characteristics, such as limited data types, dynamic variable types and properties, and extensive use of automatic type conversion
    3. Write a client-side function that uses a regular expression to validate form entry and, if an entry is invalid, displays an error message rather than submitting the form
    4. Develop event-driven programs that use HTML intrinsic event attributes, DOM event listeners, and DOM-generated events
    5. Use the DOM to modify a document's attributes and style properties as well as to modify its parse-tree representation
    6. Use de facto standard browser host objects for tasks including I/O, opening new windows, sleeping, and navigation
    7. Demonstrate the use of both a debugger and a DOM inspector in order to understand the execution of a client-side program
  3. Server-side Programming
    Topics:
    • Server-side program execution life cycle
    • Program-centric and markup-centric approaches
    • Parameter passing
    • Session
    • Cookies
    • Concurrency
    • Security issues
    • Model-View-Controller (MVC) architecture
    Learning objectives:
    1. Describe how a given web server responds to an HTTP request for a dynamic resource
    2. Discuss the strengths and weaknesses of the program-centric and markup-centric approaches to server-side development
    3. Explain parameter passing from client to server, including the generation of query strings from forms and server-side processing
    4. Describe sessions conceptually and explain how the concept can be implemented using cookies and URL rewriting
    5. Discuss the concept and implementation of cookies as well as related privacy concerns
    6. Identify and correct problems related to concurrency in server-side programs
    7. Explain common security threats such as cross-site scripting and misformed HTTP requests and demonstrate avoidance techniques for each
    8. Develop a reasonably sophisticated web application that appropriately employs the MVC architecture
  4. Web Data Communication and Processing
    Topics:
    • XML fundamentals
    • Formal definitions of XML vocabularies
    • XML namespaces
    • Client-server communication via XML documents
    • Processing an XML document
    • Web service client/server generation using higher-level tools
    • Low-level web service technologies
    Learning objectives:
    1. Recognize and correct validity errors in XML documents
    2. Discuss how XML DTDs differ from XML schemas
    3. Write documents that make appropriate use of XML namespaces
    4. Write client-server applications that communicate via XML documents
    5. Discuss ways in which an XSL transform differs from processing an XML document using a DOM API
    6. Using high-level tools, write a web service client that accesses a commercial web service
    7. Explain the meaning of given WSDL and SOAP documents
